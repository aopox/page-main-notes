# page-main-notes

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式保存，适用于展示、测试或作为备份参考。

## 项目简介

page-main-notes 是一个通用型 HTML 页面集合仓库。所有页面均为独立文件，不依赖于特定域名或后端服务。仓库结构简单，便于快速浏览和部署。

## 目录说明

```
.
├── index.html          # 首页 / 导航页（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── style.css
│   ├── script.js
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立 HTML 页面存放于此，文件名建议具有辨识度。
- `assets/`：页面共享的样式、脚本与图片资源。如页面使用独立资源，建议在页面所在目录内自行管理。

## 页面归档说明

每个 HTML 页面均为完整自包含文件，或引用 assets/ 目录下的公共资源。归档页面可能包含：

- 静态内容展示
- 功能演示组件
- 临时页面快照
- 学习或测试用例

页面不涉及任何真实业务数据或敏感信息。如需引用外部资源（如 CDN），请确保其来源可靠且长期有效。

## 维护说明

- 新增页面：在 `pages/` 下创建新 HTML 文件，若有需要可同步更新 assets/ 中的公共资源。
- 更新页面：直接修改对应 HTML 文件，建议在文件头部注释中注明修改日期与简要说明。
- 删除页面：确认页面不再需要后，移除对应文件及引用的专属资源。
- 版本控制：所有变更通过 Git 记录，建议每次提交只涉及一个页面的改动或一组相关修改。

## 部署方式

本仓库可直接通过 GitHub Pages 或其他静态托管服务部署。部署后所有页面可通过 URL 直接访问，无需额外配置。
